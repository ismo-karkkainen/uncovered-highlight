The goal is to make a TypeScript 6 package that processes v8 code coverage data along with source code and source maps to produce data that allows highlighting the uncovered parts of the code.

First, generate a TypeScript scripts which have various functions that get called and which produce code coverage data. Include throws and catches alongside normal function calls with parameters that detemine which code paths to take. Each code block should have a console.log call that indicates where the log is and whether it should be visible. The files should be UTF-8 and include multi-byte characters. Add a tsconfig that also produces a source map. Place the test scripts and related files into a subdirectory testprogs. The package.json file should have a scripts that build and run the programs so that V8 coverage data is recorded. These programs should not have any package dependencies except for typescript and packages with types as needed. When installing packages, use "npm install --save-dev --save-exact" and add only package names without version. Do not edit dependencies directly.

Second, generate a library that reads the V8 coverage data and allows mapping it to sections of the JavaScript files that were run. Use the coverage data from running testprogs and the transpiled JavaScript testprograms to add mocha/chai tests to ensure that the mappings are correct.

Third, given the output from V8 coverage data library, JavaScript script, orignal TypeScript file, and source mapping, add to the library the functionality to map the coverage data back to the TypeScript file using the source mapping. The JavaScript file might not be needed, I do not know for sure. Using the coverage data and original TypeScript sources, add mocha/chai tests that verify this stage is correct.

Fourth, coverage data for JavaScript and the corresponding file, or coverage data mapped to TypeScript file source and the TypeScript file should be combined so that we can display the uncovered parts of the source code highlighted. Produce per-line data with line number, line content, and start and end index pairs of highlighted ranges covering the line, if any. All coverage data and sources are given to the function that performs the conversion. Add mocha/chai tests to verify this part works correctly.

Fifth, a command-line tool that is given source file name, and it finds the coverage data and any source maps and transpiled JavaScript sources, and shows the given source file such that uncovered parts have been highlighted. Tool has to be able to find the coverage data and when TypeScript file is given, determine from tsconfig.json outDir where the JavaScript files and source maps are. In case the search fails, outDir and coverage directory locations can be given via optional command-line arguments. Source file or files are given after other arguments.

The tool output should optionally have line numbers prefixed to each line. Line contents are not truncated even when lines are long. Terminal escape codes are added inside the line as needed. An optional context lines argument with default limit of 2, is used to remove lines that are fully covered and are more that the limit away from line that has uncovered content. Omitted groups of lines are presented with three dots in the output when requested. Coloring for dark and light modes should be available. Default dark mode is white text and bright red highlight. Default light mode is black text and red for highlight. Add some way to specify colors or whether the highlight should be just bold text with normal text for covered parts of the source code. When multiple input files are given, output also source file name before file contents.

Also modify the README.md file to document the usage, but leave the chapter "Note from a Human" intact. Add things before and/or after that.
